# 关卡难度与相似度评测方法

本方法同时评价“有多难”和“是否只是换皮重复”。实现位于
`evaluate_levels.py`，重建流程位于 `rebuild_similar_levels.py`。

## 1. 难度：精确状态空间，不做抽样

每个状态包含所有已入场猪的尾格、头格和方向，以及各队剩余数量。
搜索规则与游戏运行时一致：两格猪连续滑行、箭头即时改向、泥坑当场停下
(再点一次继续)。

对每关完整计算：

- `min`：BFS 最短通关步数；
- `p_win`：在步数预算内均匀随机选择合法操作的精确胜率；
- `crit`：最优路线上存在“选错即无解”选项的决策点数；
- `decep`：致命错误还能继续操作多少步才暴露；
- `paths`：预算内通关操作序列数；
- `search_states`：首次找到最短解前访问的状态数；
- 猪数、最深队列及 Redirect 数量。

最终难度分为：

```text
1.35×猪数 + 0.75×最优步数
+ 3.2×(1-p_win) + 0.55×crit + 0.28×decep
+ 0.45×(最深队列-1) + 0.45×解法稀缺度
+ 0.18×log10(1+搜索状态数)
+ 0.35×Redirect数 + 0.40×Mud数
```

其中解法稀缺度为 `max(0, 3-log10(1+paths))`。最终 `levels.json` 按该分数严格
非递减排序；因此猪数可以小幅交错，不能用“猪更多”冒充“推理更难”。

## 2. 结构相似度

对两关枚举旋转 0°/90°/180°/270° 与四种镜像，取最高形状 Jaccard 相似度，
再加入队列数量构成、机制构成和棋盘规模：

```text
结构相似度 = 0.50×D4形状重合
           + 0.25×队列构成重合
           + 0.15×机制构成重合
           + 0.10×棋盘规模接近度
```

这样能识别旋转、镜像、局部挪格和只改队列数量的近似换皮。

## 3. 解题方法相似度

用 BFS 取得一条确定性的最短解，将每一步转换为不依赖坐标和朝向的方法标记：

- `Q/P`：释放队首猪，或再次移动圈内猪；
- `R/-`：本步是否经过箭头转向；
- `0/1/2/3`：滑行距离档（3 表示 3 格及以上）；
- `I/X`：本步后该猪是否完全入圈；
- `M/-`：本步是否停在泥坑里。

两条标记序列使用归一化 Levenshtein 相似度比较。它关注“先放队列、何时转向、何时
二次调整”的推理过程，不会因坐标、颜色或朝向变化而误判为新解法。

## 4. 综合阈值与重建规则

```text
综合相似度 = 0.48×结构相似度 + 0.52×解法相似度
```

任一条件成立即判为过度相似：

1. 综合相似度 `>= 0.86`；
2. 解法相似度 `>= 0.92` 且结构相似度 `>= 0.68`。

重建时不是只删重复对中的一关，而是重做所有超阈值关系的全部端点。新候选必须：

1. 保持原猪数、步数余量和机制类别；
2. BFS 可解，Redirect 必须真实改变最优解；
3. 与保留关、新生成关均不超过相似阈值；
4. D4 规范签名全局唯一；
5. 完成后重新按难度排序并全量复验。

## 5. 首轮 100 关清洗结果

- 改造前：25 对超阈值关系，涉及 30 关；
- 重做：30/30 个相关关卡全部替换；
- 改造后：0 对超阈值关系；
- 难度序列：100 关严格非递减；
- 可解性与 D4 去重：100/100 通过。

审计文件：`levels_evaluation_before.md`、`levels_evaluation_after.md`、
`rebuild_audit.json`。固定种子为 `20260717`。

## 6. 扩展到 1000 关

`expand_to_1000.py` 以清洗后的 100 关为策划种子，新增 900 个箭头关：

- 新增关全部使用解法必需的 Redirect；
- 规模从 7–8 猪逐级提高到 13–14 猪；
- 新增关全部不给步数余量；
- 每个候选在加入前与所有已接受关做同一阈值的结构/解法相似度比较；
- 合并后重新计算 1000 关难度并严格排序。

最终硬指标：

- 1000 关难度严格非递减：`5.461 → 44.565`；
- 第 701 关难度已经达到 `34.885`；
- 全部关卡不含 Gate；932/1000 关含 Redirect；
- 450 关至少双箭头、198 关三箭头；后 300 关全部含箭头，最后 100 关全部三箭头；
- 1000 关全对全比较后，超阈值相似对为 0。

提示验证由 `verify_hints.py` 完成：1000 关共复验 10,172 个最短路径状态，逐步确认
提示动作合法、提示后的解仍能在剩余步数内完成；450 个多箭头关全部覆盖。

扩展审计文件：`expansion_audit.json`。固定种子为 `20260718`。

## 7. 泥坑机制编入

`add_mud.py`(种子 `20260721`)向 L151–L900 难度带的 300 关编入 1–2 个泥坑：

- 泥坑必须改变最优解（对比无泥坑复算,装饰性泥坑被拒）；
- 保持原步数余量,`min` 与 `steps` 同步更新；
- D4 规范签名（含泥坑格）全局唯一,与全部关卡在线重查结构/解法相似度；
- 新难度分不越过 L901 分数线,保证三箭头决赛段不被稀释；
- 完成后 1000 关按含泥坑项的难度公式全局重排。

结果：泥坑首现 L121、最深 L896；293 关箭头+泥坑复合、113 关双泥坑；
超阈值相似对 0;`finale_untouched=true`。审计文件：`mud_audit.json`。
