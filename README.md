# 小猪进圈（Pigpen Puzzle）

参考短视频玩法复刻的 Godot 4 益智小游戏，本质是 **Arrow Puzzle 的反向版本**：
小猪不是滑出棋盘，而是从开口外滑进猪圈。共 **1000 关**，按精确难度递增。
（玩法机制复刻自视频，美术素材为原创绘制，未使用视频中的任何素材。）

## 规则

- 小猪有三种体型：**1 格小猪 / 2 格标准猪 / 3 格长猪**，猪圈本体最大
  **8 格宽 × 10 格高**，全部猪的身体格数恰好等于猪圈格数（要填满才算赢）。
- 猪像贪吃蛇一样**沿头走过的路径跟随**：长猪过箭头会**弯着身子**停下；
  头不能撞进自己的身体中段（正要腾出的尾格可以进）。
  小猪灵活补缝、改变奇偶性；长猪笨重，需要三格直线跑道才能安置。
- 每个开口外是一条 **FIFO 猪队**（数量不限，体型可混编）：只有**队首**的猪能被点击，
  释放后沿固定方向一路冲进猪圈，**撞到栅栏或其他小猪就停下**；开口只进不出。
- 队首进圈后，后面的猪**自动向前补位**（不耗步数）。
- 屏幕上每队最多显示 2 头，排不下的收纳成 **🐷×n** 徽章——被收纳的猪
  **不可点击**，要等前面的进圈、自己现身后才能操作。
- 每次有效移动消耗 1 步（原地被堵只会晃一下，不耗步数）；进圈的猪还能再点，
  让它往更深处滑（占位调整，同样耗 1 步）。
- **改向箭头（Redirect）**：猪头滑入蓝色箭头格后，会在同一次连续滑行中按箭头
  指向转弯。箭头关需要规划哪只猪先走这条转向路线。
- **泥坑（Mud）**：猪头滑进泥坑会**当场停下打滚**，再点一次它才继续走。
  泥坑是唯一能让猪"停在半路"的手段——用它可以故意把一只猪停在通道中间
  当挡板，代价是多花一步。箭头改变"走到哪",泥坑改变"停在哪",两者可叠加。
- **滑行轨迹预览**：把鼠标悬停（手机上按住）在可点的猪上,会显示它这一步
  经过的格子、转弯点和最终落点投影——箭头关不用再靠脑内模拟。
- **死局感知**：每走一步,游戏用精确求解器复查当前局面;一旦确认剩余步数内
  已无解,步数立刻变红、熊会站起来提醒你——不再走了好几步才发现早就输了。
  （只提示,不惩罚;撤销回到活棋后警示解除。）
- 在 **限定步数内让所有小猪全部进圈** 即过关；
  步数用完、或所有猪都动弹不得时失败——熊会扑过来抓走圈外的小猪（烟尘 + 红叉）。
- **进度自动保存**：过关后下次启动自动跳到已解锁的最高关。

## 关卡设计（1000 关难度递增）

难度**不靠抽样估计**，而是对每关的完整状态空间做精确搜索得到可复验的指标：

- **p_win** —— 均匀随机点击的**精确胜率**（对"状态 × 剩余步数"全记忆化 DP）。
  学习关要求 p_win = 100%，即**数学上证明不可能失败**；
  最难关 p_win < 0.1%，乱点几乎必死。
- **crit（关键抉择数）** —— 沿最优解路线，有多少个决策点存在"点错就注定失败"的选项。
  从第 1 关的 0 一路爬到决赛段的 15：每一步都可能万劫不复。
- **decep（欺骗深度）** —— 点出致命一步后，还能继续走几步才暴露死局。
  越深越难察觉：前期错了立刻卡死（好学），决赛段错了要走 15~16 步才发现。
  好在**死局感知**会即时示警,欺骗深度带来的是紧张感而非背叛感。
- **paths（可通关序列数）** —— 步数预算内所有能通关的点击顺序总数。
  从开局的几十种收敛到最后一关的两位数甚至个位数。

最终关卡按**精确难度分严格非递减**排列。排序综合猪数、最优步数、随机胜率、
关键抉择、欺骗深度、解法稀缺度、搜索状态数、队列深度、Redirect、泥坑与
体型混编；因此相邻猪数允许交错，不再假设“猪更多就一定更难”。完整公式见
`tools/LEVEL_EVALUATION.md`。

逐关精确指标见 `tools/levels_evaluation_after.md` 和 `tools/levels_report.txt`，运行
`python3 tools/evaluate_levels.py` 与 `python3 tools/verify_levels.py` 可全部复算。

**每一关都不一样**（三层保证，评测器与验证器强制检查）：
1. **D4 全变换去重**——每关连同它旋转 90°/180°/270° 和四种镜像共 8 个形态
   取规范签名；签名包含猪圈、队列、箭头位置/方向和门边，1000 关两两不同：
   任何一关都不是另一关转个方向或照镜子。
2. **结构相似度限制**——D4 对齐后的形状重合与队列、机制、规模共同计分，拦截
   “只挪一格”或“只改队列数量”的近似换皮。
3. **解法相似度限制**——将最短解编码为释放/二次移动、转向、距离和入圈状态
   序列，使用编辑距离拦截“棋盘不同但解题方法相同”的关卡。

猪圈候选取自矩形 / L / T / U / 十字 / 阶梯 / 缺角 / Z / H 等形状，随机骨牌
平铺后逐关求解；相似度超阈值的候选不会进入最终关卡集。

**两个难度支柱**：
1. **交叉封路**——一只猪滑入后的最终停留格，若恰好挡在另一只猪的通道上，
   那只猪就必须等它先进圈；依赖交织成约束图（构造时保证无环、即有解）。
2. **深队列**——一条队里的猪按固定顺序逐头进圈，每一头都会滑到当前能到的最深处；
   队列越深，"什么时候放这条队的下一头"越讲究，与圈内依赖叠加出更深的推演层次。

**机制教学节奏**：难度排序后 Redirect 首次出现在第 7 关；箭头数量不设为 1 个上限。
总计 932/1000 关使用箭头，其中 450 关至少双箭头、198 关三箭头；后 300 关全部
使用箭头，最后 100 关全部三箭头。验证器逐个移除箭头复算，不允许装饰性摆设。

泥坑在箭头玩法立稳后进场：首次出现在第 121 关，共 300 关（其中 293 关与箭头
复合、113 关双泥坑），分布到第 896 关为止——三箭头决赛段（后 100 关）保持纯箭头
高压。泥坑同样受"必须改变最优解"的验证约束：摆着好看的泥坑过不了验证器。

## Meta 层设计（对标成熟休闲解谜）

外围系统整体复刻休闲解谜赛道验证过的模板（Royal Match / Candy Crush /
梦幻花园系），按"离线单机可实现"取舍：

| 系统 | 参照 | 本作实现 |
|------|------|------|
| ❤ 红心体力 | Candy Crush / Royal Match | 5 心上限，失败扣 1，每 30 分钟回 1；0 心不能重试 |
| 🪙 金币 | 通用软货币 | 过关获得：基础 30 + 星级×20 + 连胜加成（上限 +50） |
| ★ 三星评级 | Candy Crush 星级 | 精确对标最优解：最优步数 3★、多 1 步 2★、过关 1★；选关面板逐关展示 |
| 🔥 连胜 | Royal Match 连胜给道具 | 连胜提升金币加成；连胜 3/5/10 时额外送随机道具 |
| 道具 | 通用三件套 | ↩ 撤销（回滚一步）· 💡 提示（实时求解器闪烁下一只该点的猪）· ➕2 步；有库存扣库存，没库存金币购买 |
| 续关 | Candy Crush "+5 moves" | 步数耗尽但未死局时报价：300🪙 +2 步继续；死局不可续（规则上救不回来） |
| 🐷 皮肤收集 | 装扮/收集 meta | 5 款小猪皮肤金币解锁，全局生效 |
| 📅 每日奖励 | 通用日活钩子 | 每天首次启动 +200🪙 和 1 个提示道具 |

其中"提示"道具直接复用与关卡生成器同一套规则的 DFS 求解器，
从**当前局面**（而非预存答案）实时求解，玩家走进死路时会明确提示
"剩余步数内已无解"，与撤销道具形成组合。

联网向系统（队伍、赛季活动、排行榜、内购）不在单机范围内，接入平台后
可在 `scripts/meta.gd` 的结算钩子上扩展。

状态全部持久化在 `user://meta.cfg`（进度在 `user://progress.cfg`）。
测试开关：`main.gd` 的 `UNLOCK_ALL`（全关解锁）与 `meta.gd` 的
`FREE_HEARTS`（红心不阻塞），发布前改回 `false`。

## 运行方式

1. 从 https://godotengine.org/download 下载 Godot 4.x（4.2+，标准版即可）。
2. 打开 Godot，「导入」选择本目录下的 `project.godot`。
3. 按 F5（或右上角 ▶）运行。过关后点「下一关」，进度持久保存。

## 项目结构

- `scripts/main.gd` — JSON 关卡读取、进度持久化、网格滑动逻辑、步数、胜负判定、UI、特效
- `scripts/pig.gd` — 小猪棋子：两格占位、滑动/受阻动画、点击区域
- `scripts/bear.gd` — 熊：角落游荡，失败时扑猪
- `scripts/pen_renderer.gd` — 猪圈绘制：地板、栅栏、开口门板、桩子（随格子大小缩放）
- `art/*.svg` — 原创美术素材
- `levels.json` — 1000 关数据（由生成器产生，精确状态空间验证）
- `tools/generate_levels.py` — 关卡生成器（逆向构造 + 机制编排 + 相似关重建 + 难度排序）
- `tools/evaluate_levels.py` — 精确难度评分、结构相似度和最短解法相似度评测
- `tools/rebuild_similar_levels.py` — 重做所有超阈值关系端点并生成审计映射
- `tools/expand_to_1000.py` — 900 关严格扩展、后段机制配额与难度门槛审计
- `tools/upgrade_multi_redirect.py` — 将后段升级为双/三箭头并在线重查相似度
- `tools/add_mud.py` — 向中后段 300 关编入解法必需的泥坑并全局重排难度
- `tools/verify_hints.py` — 验证每关提示动作合法且剩余路径能在预算内通关
- `tools/verify_levels.py` — 可解性、机制有效性、D4 去重、难度顺序和相似度终检

## 重新生成 / 验证关卡

```bash
# 重新生成（固定种子 20260716，可复现）
python3 tools/generate_levels.py

# 验证所有关卡
python3 tools/verify_levels.py

# 单独重算难度/相似度报告
python3 tools/evaluate_levels.py --report tools/levels_evaluation_after.md
```

`verify_levels.py` 逐关输出猪数、最优步数、预算、队列深度、精确指标和猪圈尺寸，
并做 D4 全查重、难度有序和超阈值相似度终检，全部通过后退出码为 0。

关卡数据格式（`levels.json` 每关）：

```json
{
  "steps": 5,
  "min": 4,
  "pen": [[x,y], ...],
  "queues": [[[cx,cy],[dx,dy],[3,2,1]], ...],
  "redirects": [[[x,y],[dx,dy]], ...],
  "muds": [[x,y], ...]
}
```

- `pen`：猪圈格子列表（本体 ≤ 8 宽 × 10 高）
- `queues`：`[圈内入口格, 朝外方向, 体长序列]`，每个开口一条。第三项为
  数组时是队首在前的体长序列（1/2/3 格）；为 int 时表示"n 头 2 格猪"（旧格式兼容）
- `steps`：步数上限；`min`：最优步数（三星评级用）
- `redirects`：多个 `[箭头格, 新方向]`；数量不限，猪头每次进入箭头格后立即改向。
- `muds`：泥坑格列表；猪头进坑当场停下，再点一次继续。与箭头不同格。

`tools/mechanize_levels.py` 会以固定配置把两个机制编入基础关卡集；直接运行
`tools/generate_levels.py` 时也会自动执行机制编排、相似关重建与难度排序，重新生成
不会丢失本轮规则。相似度方法及前后审计见 `tools/LEVEL_EVALUATION.md`。

格子大小与居中位置按"猪圈 + 可见排队槽位 + 徽章格"的包围盒自动计算，无需手调坐标。
