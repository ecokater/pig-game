# 小猪进圈（Pigpen Puzzle）

参考短视频玩法复刻的 Godot 4 益智小游戏，本质是 **Arrow Puzzle 的反向版本**：
小猪不是滑出棋盘，而是从开口外滑进猪圈。共 **100 关**，按难度递增。
（玩法机制复刻自视频，美术素材为原创绘制，未使用视频中的任何素材。）

## 规则

- 每只小猪占 **两格**，在猪圈预设开口的外面等待，朝向固定。
- 同一开口外可能**排着多头猪**：前一头不进圈，后一头就被堵着动不了；
  而且先进的会一路滑到最深处——队伍顺序本身就是谜题的一部分。
- **点击小猪**，它会沿自己的方向一路冲进猪圈，**撞到栅栏或其他小猪就停下**；开口只进不出。
- 每次有效移动消耗 1 步（原地被堵只会晃一下，不耗步数）。
- 在 **限定步数内让所有小猪全部进圈** 即过关；
  步数用完、或所有猪都动弹不得时失败——熊会扑过来抓走圈外的小猪（烟尘 + 红叉）。
- **进度自动保存**：过关后下次启动自动跳到已解锁的最高关。

## 关卡设计（100 关难度递增）

难度**不靠抽样估计**，而是对每关的完整状态空间做精确搜索得到可复验的指标：

- **p_win** —— 均匀随机点击的**精确胜率**（对"状态 × 剩余步数"全记忆化 DP）。
  学习关要求 p_win = 100%，即**数学上证明不可能失败**；
  最难关 p_win < 0.1%，乱点几乎必死。
- **crit（关键抉择数）** —— 沿最优解路线，有多少个决策点存在"点错就注定失败"的选项。
  从第 1 关的 0 一路爬到最后一关的 7~9：每一步都可能万劫不复。
- **decep（欺骗深度）** —— 点出致命一步后，还能继续走几步才暴露死局。
  越深越难察觉：前期错了立刻卡死（好学），后期错了要到 8~9 步后才发现（要提前推演）。
- **paths（可通关序列数）** —— 步数预算内所有能通关的点击顺序总数。
  从开局的几十种收敛到最后一关的两位数甚至个位数。

难度按"世界"推进——每个猪数是一个世界，世界内先给几关**学习关**（p_win=100%，
熟悉新形状与排队），再进入**陷阱关**（按难度分递增）；世界之间整体抬升。

| 世界 | 关段 | 猪数 | 学习关 | 难度分区间 | 步数余量 |
|------|------|------|------|------|------|
| ①    | 1–2   | 2 只 | 1 | 5.3 → 8.0 | +3 |
| ②    | 3–8   | 3 只 | 5 | 6.3 → 9.3 | +3 |
| ③    | 9–16  | 4 只 | 4 | 7.2 → 13.7 | +2~3 |
| ④    | 17–28 | 5 只 | 2 | 7.9 → 17.0 | +2 |
| ⑤    | 29–44 | 6 只 | 2 | 10.0 → 19.5 | +1~2 |
| ⑥    | 45–62 | 7 只 | 1 | 14.7 → 20.6 | +1 |
| ⑦    | 63–80 | 8 只 | 2 | 12.6 → 25.1 | +1 |
| ⑧    | 81–90 | 9 只 | — | 20.6 → 25.7 | +0~1 |
| ⑨    | 91–100| 10 只 | — | 22.5 → 25.9 | +0（必须最优解）|

难度分 = 猪数、依赖链深度、(1−p_win)、关键抉择数、欺骗深度、解稀缺度、排队长度的加权和
（权重见 `tools/generate_levels.py` 的 `difficulty()`），逐关数据见 `tools/levels_report.txt`，
运行 `python3 tools/verify_levels.py` 可全部复算。

同一世界内不重复同一形状签名，相邻两关也不重复——猪圈形状取自
矩形 / L / T / U / 十字 / 阶梯 / 缺角 七类，随机骨牌平铺后逐关求解验证。

**两个难度支柱**：
1. **交叉封路**——一只猪滑入后的最终停留格，若恰好挡在另一只猪的通道上，
   那只猪就必须等它先进圈；依赖交织成约束图（构造时保证无环、即有解）。
2. **圈外排队**——同一开口外排多头猪时，先点的会滑到最深处，
   队列顺序决定占位方案，与圈内依赖叠加出更深的推演层次。

## 运行方式

1. 从 https://godotengine.org/download 下载 Godot 4.x（4.2+，标准版即可）。
2. 打开 Godot，「导入」选择本目录下的 `project.godot`。
3. 按 F5（或右上角 ▶）运行。过关后点「下一关」，进度持久保存。

## 项目结构

- `scripts/main.gd` — JSON 关卡读取、进度持久化、网格滑动逻辑、步数、胜负判定、UI、特效
- `scripts/pig.gd` — 小猪棋子：两格占位、滑动/受阻动画、点击区域
- `scripts/bear.gd` — 熊：角落游荡，失败时扑猪
- `scripts/pen_renderer.gd` — 猪圈绘制：地板、栅栏、开口门板、桩子（随格子大小缩放）
- `art/*.svg` — 原创美术素材
- `levels.json` — 100 关数据（由生成器产生，BFS 验证可解）
- `tools/generate_levels.py` — 关卡生成器（逆向构造 + BFS 验证 + Monte-Carlo 难度评估）
- `tools/verify_levels.py` — 关卡验证脚本（BFS 可解性 + 包围盒 + MC 失败率）

## 重新生成 / 验证关卡

```bash
# 重新生成（固定种子 20260714，可复现）
python3 tools/generate_levels.py

# 验证所有关卡
python3 tools/verify_levels.py
```

`verify_levels.py` 逐关输出猪数、最优步数、步数预算、随机失败率和包围盒，
全部通过后退出码为 0。

关卡数据格式（`levels.json` 每关）：

```json
{
  "steps": 5,
  "pen": [[x,y], ...],
  "openings": [[[cx,cy],[dx,dy]], ...],
  "pigs": [[[tx,ty],[hx,hy],[dx,dy]], ...]
}
```

- `pen`：猪圈格子列表
- `openings`：`[圈内格子, 朝外方向]`，每个开口一条
- `pigs`：`[尾格, 头格, 前进方向]`，头格紧邻开口（或排队等候）
- `steps`：步数上限

格子大小与居中位置按关卡包围盒（猪圈 + 全部初始猪格）自动计算，无需手调坐标。
